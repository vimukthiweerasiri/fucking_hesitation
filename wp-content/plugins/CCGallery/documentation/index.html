A) Introduction - top

CCGallery WP makes it easy to embed the CCGallery HTML5 Multimedia Gallery, available at Codecanyon, in your Wordpress theme. It displays all your images, audio files and video files in a sleek and attractive interface. You get to browse the items in Coverflow mode or in a Thumbnail grid mode. You can easily switch between the two display modes and can even filter the items by displaying only images, or audio files, or video files. Filtering of items by custom categories is also supported. Also CCGallery incorporates a responsive design so that the gallery display fits in various screen sizes ranging from mobile phones to desktop browsers. The gallery features full touch-screen support which is noted especially in the Coverflow mode. CCGallery has been designed in such a way that it is extremely easy to browse through a large number of items and also with enough eye candy! CCGallery WP allows you to create multiple galleries where each gallery can have as many items as you want. Although CCGallery WP allows you to create multiple galleries you can only embed one gallery per post/page.

Please note that as of version 2.0 CCGallery utilises CSS 3D Transforms in browsers that support this feature to produce the Coverflow mode. This results in smooth animation which is especially noticeable in mobile devices such as iPhone/iPad. Currently CSS 3D Transforms are supported in Firefox 12.0+, Chrome 20.0+, Safari 5.1+, iOS 3.2+, Android 3.0+. If the browser does not support 3D Transforms then the gallery will use HTML5 Canvas for the Coverflow effect. Currently browsers that support Canvas include Firefox 3.0+, Safari 3.0+, Chrome 3.0+, Opera 9.5+ and IE9+. Internet Explorer 6, 7 and 8 do not support Canvas. Browsers which do not support Canvas will see a fallback animated effect when browsing files in the Coverflow mode. Hence visitors to your web page using old browsers will still see a nicely working gallery. The Thumbnail mode works in almost all browsers.

B) Installing the plugin - top

To install the plugin follow the steps below:

  1. Log in to the Wordpress admin panel.
  2. Head over to Plugins section and click on the “Add New” button
  3. Then click “Upload” to bring up the plugin upload dialog. Select the plugin file and upload it. Alternatively you could also unpack the plugin zip file in your computer and then upload it to /wp-content/plugins via FTP.
  4. Next go to the Plugins main area and you will see CCGallery WP being listed as one of the available plugins.
  5. Click the “Activate” button next to it and you are done.

Please note that when CCGallery WP is installed then it creates a folder called ccgallery-upload in /wp-content/uploads. So please ensure that this folder has correct permissions set. If you receive an error about the upload folder not being created then open /wp-content/uploads folder via FTP and set its permission to 777.

If you are upgrading from an earlier version then first deactivate the plugin. Then upload the new plugin files into /wp-content/plugins and overwrite the old files and after that activate the plugin. All your previously created galleries will remain intact. Some of the new options that have been introduced in the current version will have default values as required by the plugin. This will ensure that your previously created galleries keep on working. However it is best recommended that you check all your galleries and see if they are working as you want. You can also tweak the values of the new options in the Settings page to your liking for a particular gallery. If by any chance you find that the fields for the new option parameters in the gallery settings page are empty after updating the plugin, then deactivate the plugin, logout of WP Dashboard, login again and then activate the plugin. After that the new option parameters will have the default values in all your saved galleries.

C) How to Use - top

After activating the plugin you will see that a new menu item has been created on the left called CCGallery. This menu item has two sub menus - Gallery Manager and Create New Gallery. These names are self explanatory and whenever you want to create a new gallery you will head over to the Create New Gallery page, and all your existing galleries will be listed in the Gallery Manager page, from where you can edit or delete galleries.

So first of all to create a new gallery head over to the appropriate page and you will be presented with a list of options. They are described below:

Gallery name

You must provide a name to the gallery that you are going to create. This is a required field and you will receive an error if you leave it empty. Although different galleries can have the same name but it is advisable that you provide unique names so that it is helpful for you to quickly differentiate between galleries.

Gallery description

You can write a description for the gallery here. This field is optional.

Gallery Options

A list of options that are applicable to the gallery. Their functions are described as follows:

  • Colour Scheme - Two colour schemes are available. Choose one from the available list.
  • Starting mode - Choose whether the Coverflow or the Thumbnails mode will be shown when the page first loads.
  • Fade edges - By default the edges of the coverflow images section are faded out. You can disable this if you want, which has the advantage that you can place the gallery in a container with an image or gradient background. If this option is enabled then the gallery can only be placed in a container with a solid colour background.
  • Coverflow Background - Mention the colour value (in hex or rgb) that will be used as the background for the coverflow. This is required to set up the faded effect at the edges. By default the background colour of the body element is used. Please note that if this field is left empty and also your body element has no background colour defined then a default colour of black will be used.
  • Coverflow Start Item - You can mention the number of the item that will be highlighted in the Coverflow when the page first loads. If this is empty then the middle-most item will be highlighted.
  • Disable Reflection in Coverflow - You can choose to disable the reflection of the coverflow images.
  • Open Links in New Windows - You can choose to open the links associated with gallery thumbnails in a new window/tab instead of the same window/tab as the gallery (which is the default option).
  • Detect Mobile Devices - You have the option of detecting mobile devices and serving them low resolution videos, which you can upload below.
  • Autoplay Video and Audio - Enable this to start playing video/audio files automatically when the media player shows up. This setting will also affect Youtube/Vimeo videos.
  • Play in a Loop - Enable this option to play video/audio files continuously in a loop. This setting will not affect Youtube/Vimeo videos.
  • Store Volume Level - Enable this option to retain the volume level of the player when it is closed and volume will be set to that level when the player is opened again. This setting will not affect Youtube/Vimeo videos.
  • Custom Categories - Enter a comma separated list of custom category names that will be displayed in the filter menu bar at the top in the gallery.
  • Show File Type Buttons - Enable this option to show file type buttons (photo, audio, video) in the filter menu. If you have custom categories then you may not need to show these buttons.
  • Hide Gallery Menu - You can choose to hide the gallery menu completely.
  • Starting Category - Choose the starting category whose items will be shown when the page first loads.
  • Coverflow Mode Image Size - Set the width and height (in px) of the images used in the Coverflow section.
  • Thumbnails Mode Image Size - Set the width and height (in px) of the images used in the Thumbnails section. If your uploaded image is such that it cannot be scaled and cropped to the entered size then you will see that, that item in the thumbnail grid has a different size than the other items. In this case you either have to upload a different image or edit the gallery css file and edit the dimensions of #thumbgallery li and #thumbgallery img according to your required size.
  • Hide Coverflow Mode - If you don't want the Coverflow mode then you can enable this. Please note that for this to take effect you will also have to set the starting mode to Thumbnails. When this is enabled then the View mode buttons in the gallery menu bar will also be hidden.
  • Hide Thumbnails Mode - If you only want the Coverflow mode then you can enable this to hide the Thumbnails section. On enabling this the View mode buttons in the gallery menu bar will also be hidden. Please take care to have the Starting mode as Coverflow (which is the default option) in order for this to take effect.
  • Hide Coverflow Item List - If you only want the images in the Coverflow section but not the list that appears below the images then you can enable this to hide that list.

Gallery Items

Here you can create the items that will be displayed in the gallery. The items that you create can also be sorted into any order you want by dragging them. The configuration options for the items are described below.

  • File Type - Choose the file type of the item, i.e. whether it is a photo, audio or video.
  • Category - Enter a comma separated list of category names to which this item belongs. The list of category names entered here should be present in the list of custom categories entered earlier.
  • Image file - Upload an image file which should be used to generate the images for the Coverflow and Thumbnails mode. Also if the file type is photo then this image file will also be displayed in the Lightbox.
  • Audio Files - If your file type is audio then you will be given the option to upload mp3 and ogg formats of your audio file. If you only upload the mp3 format then in browsers that don't support it natively, like Firefox, Flash will be used to play that file.
  • Video Files - If your file type is video then you will be given the option to upload the mp4, webm and ogv formats of your video file. If you chose to detect mobile devices then you are given the option to upload a low resolution mp4 video file specifically for mobile devices. If you only upload a mp4 file then Flash will be used to play it in browsers that don't support it natively.
  • Youtube Video - If you chose video file type then you can enter a Youtube video link instead of uploading files. The link should be of the form http://www.youtube.com/watch?v=xxxxxxxxxxx. Any other form of the link won't work.
  • Vimeo Video - If you chose video file type then you can enter a Vimeo video link instead of uploading files. The link should be of the form http://vimeo.com/12345678. Any other form of the link won't work.
  • Title - Enter the title for the gallery item, which will be displayed in the Coverflow and Thumbnail modes and also in the Lightbox.
  • Caption - Here you can write a short description of the gallery item. This field can contain any html code. It will be displayed in the Coverflow and Thumbnail modes and also in the Lightbox, but if it is too long then it will get cut off in the Coverflow and Thumbnail modes, but will be displayed fully in the Lightbox.
  • Link - If you would like the thumbnails of the gallery items (both in Coverflow and Thumbnail modes) to act as links to external pages then you can enter the url of that link here.

Whenever you want to create a new gallery then the form on that page will be filled out with default option values, so that you only have to create the gallery items in order to make your gallery work. You can just change the settings that you want to customise and save the gallery settings. Needless to say, whenever you are going to edit an existing gallery, the same page will open and all the form fields will be filled out with the settings with which you created the gallery.

If you want to delete an uploaded file then just clear out the text field for that file and save your settings and the file will be deleted. When you upload a file then it is uploaded immediately to your server but in order to make the plugin remember the file you have to save the settings. On the other hand the file is deleted only if you clear the text field and then save the settings. When you delete an entire gallery from the gallery manager page then all the files associated with that particular gallery will also be deleted.

Multimedia files

As already stated CCGallery supports images, audio and video. In case of images if the dimensions of the large version of the image is bigger than the browser window dimensions then that image will be resized to fit inside the browser window. For playing audio and video CCGallery uses Mediaelement.js. This is a HTML5 audio and video player. For browsers that do not support HTML5 audio or video playback, Mediaelement.js enables fallback to Flash and Silverlight. Now different browsers require different file formats for native playback of HTML5 audio and video. For audio you must upload a mp3 version of the file which is supported by Safari, Chrome and IE9. You must upload an ogg version if you want to have native playback in Firefox and Opera. The ogg format is also supported by Chrome. But you can upload only the mp3 version and Mediaelement.js will use Flash to play the audio file in Firefox and Opera. Similarly for the case of video playback, if you want to have a wide range of browsers natively play the video, then you must upload mp4, webm and ogv versions of the video. But here also you can only upload a mp4 version (h.264) and Mediaelement.js will use Flash or Silverlight to play the video in browsers that do not support that format. If you use only mp4 (h.264) videos and find that when there is Flash fallback in unsupported browsers, the entire video needs to be downloaded before it can be watched, then the index of the mp4 file is placed at the end. To remedy this case use the utility QTIndexSwapper on your mp4 files. If you find that after uploading CCGallery to your server your browser is not able to play video files then you may need to configure your server in order to enable it to stream video files. For an Apache server place the following code in an .htaccess file and place it at the root of your website. 

AddType video/ogg .ogv
AddType video/mp4 .mp4
AddType video/webm .webm

Keyboard Shortcuts

CCGallery also provides some handy keyboard shortcuts to facilitate easier browsing. Most of these shortcuts are however effective only in the Coverflow mode. When in the Coverflow mode you can press LEFT and RIGHT keys to flip through the images. This also of course highlights the appropriate item in the list below the images. When a particular item is highlighted you can then press ENTER to open the modal overlay box. In order to close the overlay you can then press ESC. Note that this last shortcut also works when the overlay is opened from the Thumbnails section. Also if the thumbnails are made to behave as links then on pressing ENTER, when an item is highlighted, will load the page pointed to in the link url.

PHP Upload Settings

CCGallery allows to upload files of sizes upto 100MB. But your server configuration most probably won't allow large sized file uploads by default and in most cases the deafult upload size limit is set at 2MB. In order to upload large sized files you have to edit your php.ini file (contact your hosting provider if you can't find it) and change the values for the following two rules: 

post_max_size = 100M
upload_max_filesize = 100M

Here we have set the maximum upload file size to be 100MB. But uploading large files also requires sufficient memory and you have to set an appropriate memory limit for PHP to use. For most regular sized file uploads you can edit the following rule in php.ini 

memory_limit = 64M

Also uploading large files require time and most users won't have super-fast internet connections, so you also have to set the input time for php scripts to an appropriate value. In the following example we are changing the input time to be 30 minutes. 

max_input_time = 1800

In the above example we have entered time in seconds. Sometimes you may also need to change the execution time of the php scripts which is affected by the rule called max_execution_time where also the time is entered is seconds. If you are unsure about editing your php.ini file or have no idea what it is then it is best to contact your hosting provider.

Responsive Design

CCGallery features a responsive design and thus can adapt itself to be viewed in desktop browsers as well as in mobile devices. As of version 2.0 CCGallery features a fluid design and will therefore also change its width if you place it in any container of a particular width. The default width of the gallery (div#ccgallery) is 1000px, which changes as the browser viewport width changes or if the parent container has a smaller width, and has a minimum value of 372px. To have the best effect for the responsive design the gallery should be in a page all by itself and as a body (root) level element, meaning that it should not be placed inside any container. That way the responsive design of the gallery will function fully when the browser width changes. You can see this effect by opening the demo of the gallery in Codecanyon in your browser and resizing its width.

D) Embed galleries - top

All the galleries that you create will be visible in the Gallery Manager page. When you go to that page you will see that each gallery has a field in the table titled shortcode. In this field you will see the shortcode that is required for embedding a particular gallery. Just copy the shortcode for a gallery, that is displayed there and then head over to Posts section in Wordpress. Open any post or page where you want to have a gallery, in the Wordpress editor, and then paste the shortcode at a place where you find suitable, and then save or publish that post or page. Then open that post/page in your browser and you will see the gallery in action with all the settings that you created it with. If you want to directly write the embed code in a Wordpress template file then you will have to use the do_shortcode function. Using this function has the advantage that you can embed a gallery in a part of the Wordpress template where content is not pulled using the Wordpress Loop. An example usage of this function to embed a gallery is: 

    <?php echo do_shortcode('[ccgallery id="1"]'); ?>

Special Notes for IE

IE8 and below needs two javascript files to be declared in the header of your WP theme, in the <head>, in order to make it recognize new HTML5 tags and also to be more standards compliant to make the entire gallery work correctly. You should declare the two files in the header in the following way: 

<!--[if lt IE 9]>
	<script src="http://html5shiv.googlecode.com/svn/trunk/html5.js"></script>
	<script src="http://ie7-js.googlecode.com/svn/version/2.1(beta4)/IE9.js"></script>
<![endif]-->

IE also needs the CSS3 PIE file in order for modern CSS3 styles that this gallery uses, to be rendered there. You must place the PIE.htc file that is present in the css folder of the plugin, in the root of your site so that it is loaded in IE. Also IE9 is very strict regarding mime-types, so if you find that the htc file is not being loaded in IE9 then you must set the correct mime-type as 

AddType text/x-component .htc
